# Платформа развития ТСЧ — Техническая документация

## 1. Обзор решения

**Платформа развития ТСЧ** — корпоративное веб-приложение для управления процессами открытия и развития торговых точек сети X5 Group. Платформа объединяет проектное управление, строительную аналитику, календарное планирование, новостную ленту и систему порталов для сотрудников направления развития.

### 1.1 Бизнес-цели

- Единая точка входа для всех процессов открытия магазинов
- Персонализированный рабочий стол с виджетами для каждой роли
- Контроль строительных проектов с оценкой рисков (АСКО)
- Управление календарём и загрузкой сотрудников
- Внутренние коммуникации через новостную ленту и опросы
- Быстрый доступ к корпоративным порталам и системам

### 1.2 Целевая аудитория

| Роль | Код | Описание |
|------|-----|----------|
| Администратор | `admin` | Полный доступ ко всем сервисам, управление контентом |
| Менеджер развития | `expansion_manager` | Поиск и запуск объектов, оценка локаций |
| Руководитель стройки | `construction_manager` | Контроль строительства, стройконтроль |
| Управляющий недвижимостью | `property_manager` | Эксплуатация и арендные договоры |

### 2.3 Принципы архитектуры

1. **API-first** — фронтенд работает исключительно через REST API
2. **Конфигурация в БД** — все настройки, метаданные и справочники хранятся в PostgreSQL, не в коде
3. **Бизнес-логика на бэкенде** — агрегация, фильтрация, расчёты выполняются на сервере
4. **Ролевая модель** — JWT-аутентификация с role-based access control
5. **Персонализация через БД** — настройки пользователя (тема, layout, избранное) сохраняются на сервере

## 6. Безопасность

### 6.1 Аутентификация

- **Алгоритм**: PBKDF2-SHA256, 100 000 итераций
- **JWT**: HS256, срок жизни 4 часа (настраивается через `access_token_expire_minutes`)
- **Хранение токена**: `localStorage` (ключ `platform_token`)
- **Secret key**: из переменной окружения `SECRET_KEY`

### 6.2 Авторизация

- Role-based access control (RBAC)
- Декоратор `require_roles` для защиты эндпоинтов
- Административные операции (новости, опросы) только для роли `admin`

### 6.3 CORS

- Origins из `settings.cors_origins` (конфигурируется через env)
- Дополнительные origins через `CORS_EXTRA_ORIGINS`
- Статические файлы (uploads) с отдельным CORS-middleware

## 8. Бизнес-требования

### 8.1 Функциональные требования

#### FR-1: Рабочий стол с виджетами
- Персонализируемый рабочий стол с drag-and-drop
- Виджеты: АСКО, Новости, Календарь, Активность, Лента активностей, Задачи
- Три режима масштаба (1/3, 2/3, 3/3 колонки)
- Видимость виджетов настраивается пользователем
- Layout сохраняется в БД

#### FR-2: Строительная аналитика (АСКО)
- Виджет со стадиями строительных проектов
- Карточки объектов с прогрессом и рисками
- Drill-down в раздел «Стройка» с фильтрами по стадии
- Данные из PostgreSQL, не хардкожены

#### FR-3: Календарь
- Месячный вид с точками событий
- Дневной вид с 30-минутными слотами (8:00–21:00)
- Множественные события на один слот (как в Outlook)
- CRUD событий через API
- По умолчанию свёрнут

#### FR-4: Порталы
- Каталог корпоративных систем с категориями
- Поиск по названию и фильтрация по категории
- Три режима отображения: сетка, компактный, список
- Избранные порталы (сохраняются в БД)
- Горизонтальный слайдер на рабочем столе (опционально)

#### FR-5: Новости и опросы
- Создание/редактирование/публикация новостей (admin)
- Опросы с голосованием
