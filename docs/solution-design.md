# Архитектура решения (MVP)

## Целевой контур

- Единое рабочее место для команд развития: поиск объекта, строительство, запуск, управление недвижимостью.
- Backend-центричная логика: frontend только отображает данные и отправляет пользовательские действия.
- Ролевая модель и быстрый вход по ролям для оперативной работы.

## Принципы (best practices)

- API-first подход: frontend работает только через контрактные REST endpoints.
- Thin frontend / fat backend: бизнес-правила, проверки и агрегации находятся в Python-сервисах.
- Персонализация через сервер: состояние виджетов хранится в БД per-user.
- Конструктор рабочего стола по UX best practices:
  - Явный `Edit mode` вместо скрытых inline-изменений.
  - Четкие действия `Save / Cancel / Reset`.
  - Черновик изменений до сохранения, чтобы исключить случайные потери.
  - Контролируемая сложность: ограниченное число колонок и размеров.
  - Доступность: читабельные подписи и фокус на понятные элементы управления.
- Безопасность:
  - JWT для сессии пользователя.
  - Ролевые ограничения через dependency layer.
  - CORS ограничен origin фронта.
- Наблюдаемость:
  - `/health` endpoint для проверки доступности.
  - Структура данных готова к подключению аудита и метрик.

## Компоненты

- `frontend` (React + Vite)
  - Авторизация (обычный вход + quick access).
  - Личный кабинет с виджетами и метриками.
  - Каталог бизнес-систем с переходом по клику.
  - Табличный вид pipeline открытия магазинов.
- `backend` (FastAPI + SQLAlchemy)
  - Роуты: `auth`, `users`, `dashboard`, `catalog`, `processes`.
  - Seed-инициализация тестовых данных при первом старте.
- `postgres` (локальный инстанс проекта)
  - Пользователи/роли, виджеты, метрики, бизнес-системы, проекты.

## Как масштабировать дальше

- Выделить доменные микросервисы:
  - `site-selection-service`
  - `construction-service`
  - `opening-readiness-service`
  - `property-management-service`
- Добавить workflow engine (BPMN) для согласований.
- Вынести auth в SSO/OIDC и ввести тонкую permission-модель на операции.
- Внедрить event-driven интеграции (Kafka/RabbitMQ) с ERP/HR/финконтуром.
